# Agent Governance

## Purpose

This document explains how Aodcast should be developed and maintained by multiple agents over time.

## Governance Principles

- repository rules are part of the architecture
- cross-agent coordination must be explicit
- shared contracts are protected assets
- documentation must evolve with implementation
- periodic maintenance is required, not optional

## Canonical Governance Files

- [AGENTS.md](../../AGENTS.md)
- [subagents.md](subagents.md)
- [maintenance-playbook.md](maintenance-playbook.md)

## Change Rules

Update governance docs whenever changes affect:

- repo structure
- code ownership boundaries
- shared schemas
- interview flow
- provider extension model
- cleanup workflows
